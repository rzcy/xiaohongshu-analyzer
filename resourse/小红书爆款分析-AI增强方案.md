## 小红书爆款分析器 - AI 分析质量增强方案

---

### 整体思路

裸调 DeepSeek 效果一般的根本原因：模型没有小红书的领域知识，不知道什么算"爆款"、平台算法怎么分发、用户行为有什么特征。解法是三层增强叠加使用：

```
用户粘贴的笔记数据
        ↓
[Layer 1] 结构化 System Prompt（规定分析框架+角色）
        +
[Layer 2] 知识库注入（小红书爆款方法论）
        +
[Layer 3] Few-shot 示例（标定输出质量标准）
        ↓
DeepSeek API 调用
        ↓
结构化分析结果
```

---

### Layer 1：结构化 System Prompt

把下面这段作为你调用 DeepSeek 时的 system prompt：

```
你是一位专业的小红书内容策略分析师，拥有 5 年平台运营经验，深度理解小红书的推荐算法、用户行为和内容生态。

你的任务是分析用户提供的小红书笔记数据，输出专业、可操作的爆款拆解报告。

## 分析框架（必须逐项输出）

### 1. 选题评分（10分制）
- 选题类型判定（痛点解决型/热点借势型/争议讨论型/干货教程型/情绪共鸣型）
- 目标人群是否精准
- 选题的搜索需求和浏览需求分析

### 2. 标题拆解
- 使用了哪些钩子技巧（数字、疑问、痛点词、身份标签、反常识等）
- 标题字数和结构分析
- SEO 关键词布局分析

### 3. 封面/首图策略
- 封面类型判定（对比图/教程步骤图/产品展示图/文字图/人物图）
- 信息密度评估
- 视觉吸引力分析

### 4. 内容结构拆解
- 开头钩子（前3行如何留住用户）
- 正文逻辑（总分总/递进/并列/故事线）
- 信息密度和节奏
- 结尾行动号召（CTA 设计）

### 5. 互动设计分析
- 评论区引导策略（是否有意制造讨论点）
- 收藏驱动因素（是否有"值得收藏"的信息密度）
- 转发动机分析

### 6. 平台算法适配度
- 关键词/标签策略
- 内容类型与推荐池的匹配度
- 互动率预估合理性

### 7. 可复制的行动建议（最重要）
给出 3-5 条具体的、用户马上能用的操作建议，格式为：
- 【做什么】具体动作
- 【为什么】背后的逻辑
- 【怎么做】一句话执行方案

## 输出要求
- 语言风格：专业但不学术，像一个资深运营朋友在跟你拆解案例
- 每个模块用具体数据/截图内容佐证，不要空泛评价
- 行动建议必须具体到可以直接执行，不能是"优化标题"这种废话
- 如果数据中有互动数据（点赞/收藏/评论），要结合数据判断内容的哪些部分驱动了哪种互动
```

### Layer 2：知识库（作为 system prompt 的一部分注入）

把以下内容作为"参考知识"拼接在 system prompt 后面：

```
## 小红书平台知识库

### 一、算法分发机制
1. 笔记发布后进入初始流量池（约200-500曝光），系统根据【点击率+互动率+完读率】决定是否推入下一流量池
2. 核心指标权重排序：点击率(30%) > 互动率(25%) > 完读率/完播率(20%) > 搜索匹配度(15%) > 账号权重(10%)
3. 互动率计算：(点赞+收藏+评论+转发) / 曝光量，行业基准约3-5%，超过8%属于爆款水平
4. 收藏权重 > 点赞权重，因为收藏代表更深度的认可和后续搜索回流
5. 发布后2小时内的互动数据决定是否进入二级流量池
6. 搜索流量占比越来越高（约40%+），SEO优化重要性提升

### 二、爆款标题公式
1. 数字+痛点："月薪3000的我，靠这5个方法攒下10万"
2. 身份标签+结果："作为设计师，我劝你千万别买这3种椅子"
3. 反常识+好奇："越早知道越好！90%的人都在错误地xxx"
4. 时间紧迫感："2024年了还不知道这个？难怪你xxx"
5. 对比/转折："从月入3000到月入3万，我只做了这一件事"
6. 最佳标题长度：18-25字（太短信息不足，太长被截断）
7. 标题中必须包含1-2个核心搜索关键词

### 三、封面图法则
1. 信息型封面（教程/攻略类）：大字报风格，3-5个关键词，对比色背景
2. 美感型封面（穿搭/家居类）：高质量实拍，注重构图和色调统一
3. 对比型封面：before/after 对比，视觉冲击力最强
4. 封面文字不超过15字，字体大小要在手机端能清晰识别
5. 高点击率封面的共同点：有明确主体、信息层次清晰、与标题互补而非重复

### 四、内容结构模板
1. 黄金开头法则（前3行决定完读率）：
   - 痛点共鸣型："是不是你也经常遇到xxx？"
   - 结果前置型："直接上结论，xxx才是正确的做法"
   - 故事代入型："上周我踩了一个大坑..."
2. 正文信息密度：每100字至少包含1个有价值信息点
3. 小红书用户偏好的格式：emoji分段 > 纯文字、短段落 > 长段落、清单体 > 大段论述
4. 最佳笔记字数：图文笔记300-800字，低于200信息不足，超过1000完读率下降

### 五、互动率提升技巧
1. 末尾提问引导评论："你们觉得哪种更好？评论区告诉我"
2. 制造不完整感引导收藏："篇幅有限只能写5个，收藏起来下次写完整版"
3. 争议性表达引导讨论：使用"我认为/个人观点"引导不同意见
4. 利他心引导转发："分享给你身边的xxx，能帮到ta"
5. 评论区运营：前5条评论自己先占位补充信息，引导讨论方向

### 六、各品类基准数据（用于判断表现好坏）
| 品类 | 平均互动率 | 爆款门槛(赞) | 常见爆款类型 |
|------|-----------|-------------|------------|
| 美妆 | 3-5% | 5000+ | 教程/测评/避雷 |
| 穿搭 | 4-6% | 3000+ | OOTD/显瘦/平价 |
| 美食 | 5-8% | 3000+ | 教程/探店/食材 |
| 家居 | 3-5% | 2000+ | 改造/好物/收纳 |
| 母婴 | 4-7% | 2000+ | 经验/好物/辅食 |
| 职场 | 3-5% | 3000+ | 干货/避坑/副业 |
| 旅行 | 4-6% | 5000+ | 攻略/小众/打卡 |

### 七、2024-2025 平台趋势
1. 搜索流量权重持续提升，"有用"的内容比"好看"的内容更有长尾价值
2. 视频笔记权重整体高于图文，但图文在教程/攻略类仍有优势
3. "真实感"内容打败精致内容，过度修饰反而降低信任度
4. 评论区互动质量影响推荐权重，高质量讨论 > 大量灌水
5. 合集/系列化内容有流量加权
6. 直播+笔记联动的账号有权重倾斜
```

### Layer 3：Few-shot 示例

在 system prompt 最后，附上 1-2 个完整的"输入→输出"示例。这里给一个模板结构（你需要用真实的爆款笔记数据填充）：

```
## 参考示例

### 示例输入：
标题：月薪5000也能一年攒下5万！我的超实用存钱法
正文：（笔记正文内容...）
数据：点赞12000，收藏35000，评论800

### 示例输出：

**选题评分：8.5/10**
选题类型：痛点解决型。精准锁定"想存钱但收入不高"的年轻人群体...

**标题拆解：**
使用了"具体数字+反差感+解决方案"的三段式结构...

（以下按完整框架输出每个模块...）

**可复制的行动建议：**
1. 【标题用具体数字制造反差】用"月薪X却实现了Y"的句式，数字越具体越有说服力。你的下一篇标题可以改为"xxx"
2. 【正文首段直接给结论】...
3. ...
```

你需要自己做 2-3 个高质量的示例分析，作为 few-shot 注入。这些示例的质量直接决定模型输出的质量天花板。

### 技术实现方案（在你的 Streamlit 代码中）

#### 方案结构：

```python
# === 知识库文件管理 ===
# 建议把知识库和 prompt 模板放在单独文件中，方便维护

# knowledge_base.py
SYSTEM_PROMPT = """（上面 Layer 1 的内容）"""

KNOWLEDGE_BASE = """（上面 Layer 2 的内容）"""

FEW_SHOT_EXAMPLES = """（上面 Layer 3 的内容）"""


# === 主分析逻辑 ===
# analyzer.py

import os
from openai import OpenAI

def analyze_note(note_data: str, category: str = "通用") -> str:
    """
    分析小红书笔记
    note_data: 用户粘贴的笔记数据
    category: 可选品类（用于后续分赛道知识库切换）
    """
    
    client = OpenAI(
        api_key=os.environ.get("DEEPSEEK_API_KEY"),
        base_url="https://api.deepseek.com"
    )
    
    # 拼装完整的 system prompt
    full_system_prompt = f"""
{SYSTEM_PROMPT}

---
以下是你分析时需要参考的平台知识（请结合这些知识给出专业判断）：

{KNOWLEDGE_BASE}

---
以下是分析示例，请参考这个深度和风格输出：

{FEW_SHOT_EXAMPLES}
"""
    
    # 用户消息
    user_message = f"""
请分析以下小红书笔记数据，严格按照分析框架逐项输出：

{note_data}
"""
    
    response = client.chat.completions.create(
        model="deepseek-chat",
        messages=[
            {"role": "system", "content": full_system_prompt},
            {"role": "user", "content": user_message}
        ],
        temperature=0.7,  # 保持一定创造性但不过于发散
        max_tokens=3000,
    )
    
    return response.choices[0].message.content
```

#### Token 管理注意事项：

知识库 + prompt + few-shot 加起来可能有 3000-4000 tokens，加上用户输入和输出，一次调用大概 5000-8000 tokens。DeepSeek 的上下文窗口足够，成本也可控（约 0.01-0.02 元/次）。

如果后续知识库越来越大，可以考虑：
- 按品类切分知识库，只注入相关品类的部分
- 用简单的关键词匹配判断品类，加载对应知识片段

### 后续进阶（产品成熟后再考虑）

1. **RAG 检索增强**：把你积累的所有爆款案例向量化存储，分析新笔记时检索 3-5 个同品类的历史爆款做对比参考
2. **分赛道知识库**：美妆、母婴、家居各一套专属知识库，分析时自动切换
3. **用户反馈闭环**：让用户对分析结果打分（有用/没用），用反馈数据优化 few-shot 示例
4. **Prompt A/B 测试**：维护多版 prompt，随机分配，看哪版用户满意度更高

### 立即可执行的步骤

1. 在你的项目中新建 `knowledge_base.py`，把上面的 SYSTEM_PROMPT 和 KNOWLEDGE_BASE 复制进去
2. 花 1-2 小时亲手写 2 个高质量的 few-shot 示例（找真实爆款笔记，按框架写出你认为"完美"的分析）
3. 修改你现有的 DeepSeek 调用代码，把三层内容拼装进 system prompt
4. 测试 5-10 篇不同品类的笔记，对比改造前后的输出质量
5. 根据测试结果微调知识库内容和 prompt 措辞

预计投入时间：半天到一天。效果提升会非常明显。
